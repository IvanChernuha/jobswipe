# Changelog — 2026-03-30

## Bugs Fixed

### CRITICAL: Gemini API key leaked in error responses
- **File:** `backend/app/routers/cv.py`
- When LLM calls failed (e.g. 429 rate limit), the raw error including the full API key was returned to the frontend
- Fixed: errors are now logged server-side, generic message sent to client

### CRITICAL: `bulk-jobs` and `extract` endpoints crash (500)
- **File:** `backend/app/routers/cv.py`
- Both endpoints queried `employer_profiles.id` which doesn't exist (table uses `user_id` as PK)
- Fixed: changed to `select("user_id")` and updated all references

### JSON truncation bug — tags and location missing from parsed jobs
- **Files:** `backend/app/services/llm/gemini.py`, `backend/app/services/llm/deepseek.py`
- `maxOutputTokens` was 2048 — Gemini would write a long description, run out of tokens, and the JSON got cut off mid-response
- Only `title` and `description` survived; location, tags, salary were all lost
- Fixed: increased `maxOutputTokens` to 8192 and reordered the LLM prompt so tags come BEFORE description (if truncation happens, description is lost, not tags)

### File encoding bug — Hebrew text garbled
- **File:** `backend/app/services/cv_parser.py`
- Plain text files were always decoded as UTF-8, breaking Windows-1255 Hebrew files
- Fixed: added `chardet` library for auto-detection of file encoding

### Tags not visible after job creation (required page reload)
- **File:** `frontend/src/pages/Jobs.tsx`
- When adding a newly created job to state, the code hardcoded `tags: []`, overwriting the tags returned by the API
- Fixed: removed the `tags: []` override so API response tags are preserved

## Features Added

### `min_experience_years` field on job postings
- **Database:** added `min_experience_years integer` column to `job_postings`
- **Backend models:** added to `JobPostingCreate`, `JobPosting`, `JobPostingUpdate`, `JobPostingWithStats`
- **LLM prompts:** updated Gemini and DeepSeek prompts to extract min experience years
- **Frontend:** added number input in job form, pre-filled from LLM extraction

### Vertex AI LLM provider (uses Google Cloud $300 credits)
- **New file:** `backend/app/services/llm/vertex.py`
- Same Gemini models but billed through Google Cloud instead of AI Studio
- Authenticates via service account or Application Default Credentials
- Added to factory pattern — switch with `LLM_PROVIDER=vertex` in `.env`

### LLM provider factory now supports 3 providers
- `gemini` — Google AI Studio (API key)
- `vertex` — Google Cloud Vertex AI (service account / ADC, uses $300 credits)
- `deepseek` — DeepSeek API
- Switch between them with one env var change, no code changes

## LLM Prompt Improvements
- Temperature set to 0.0 for deterministic results (was 0.1)
- Fields reordered: tags come before description to survive truncation
- Description limited to 50 words / 1-2 sentences
- Added multilingual support note ("text may be in any language, respond in English")
